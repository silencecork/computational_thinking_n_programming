條件式步驟¶

比較運算子 Comparison Operators¶

  • 布林(Boolean)表達式問一個問題並產生是或否結果，我們用它來控製程序流程
  • 使用布林表達式的比較結果為True / False 或 Yes / No
  • 比較運算子會查看變數內容，但不更改變數的內容
  • <
    • 小於
  • <=
    • 小於等於
  • ==
    • 等於
  • >=
    • 大於等於
  • >
    • 大於
  • !=
    • 不等於
In [ ]:
x = 5
if x == 5:
  print('等於 5')
if x > 4:
  print('大於 4')
if x >= 5:
  print('大於等於 5')
if x < 6:
  print('小於 6')
if x <= 5:
  print('小於等於 5')
if x != 6:
  print('不等於 6')

大於 4

縮排 Indentation¶

  • 在if或for之後增加縮排
  • 保持縮排指明if或for程式碼區塊的範圍（哪些行受if / for影響）
  • 將縮排減少到if語句或for語句的級別，以指示區塊(Block)的結尾
  • 空行(Blank Lines)將被忽略，不會影響縮排
  • 註釋(Comments)不論縮排或是不縮排都是被忽略

警告：不要使用Tab縮排¶

  • 使用Tab縮排與使用多個空白鍵進行縮排外表看起來相同
  • 但混用在同一個程式碼中時就會出現「indentation errors」
  • 在Colab中編輯程式碼時會自動調整
In [ ]:
x = 5
if x > 2 :
    # 增加縮排，表示是if要執行的內容
    print('Bigger than 2')
    print('Still bigger')

# 跳出if，文字要縮排回跟if相同的位置
print('Done with 2')

# 這是沒有縮排
for i in range(5) :
    # 這是第一層縮排
    print(i)
    if i > 2 : 
        # 這是第二層縮排
        print('Bigger than 2')
    # 恢復到第一層縮排
    print('Done with i', i)
# 恢復到沒有縮排
print('All Done')

Bigger than 2
Still bigger
Done with 2

雙向條件式 Two-way Decision¶

  • 有時我們想在判斷式為true的情況下做一件事，而在false的情況下做另一件事
  • 就像路上的叉子-我們只能選擇其中一條路徑，但不能全部選擇

try / except 結構¶

  • 您用try和except包圍代碼的危險部分
  • 如果在try中的程式碼是沒問題的 - except的部分則會忽略
  • 如果在try中的程式碼出錯了 - 程式就會跳向except的部分
In [6]:
astr = 'Hello Bob'
istr = int(astr)
print('First', istr)
astr = '123'
istr = int(astr)
print('Second', istr)

程式補上try/except¶

In [7]:
astr = 'Hello Bob'
# 當第一段轉換失敗時，程式碼就會跳到except
# 且程式碼正常繼續執行
try:
    istr = int(astr)
except:
    istr = -1

print('First', istr)

astr = '123'
# 當第二段轉換成功。程式並不會進入except
# 程式碼正常繼續執行
try:
    istr = int(astr)
except:
    istr = -1

print('Second', istr)

First -1
Second 123

try/except示意圖¶

  • 放在try之中的程式碼盡量少，因為一旦出錯接下來的程式碼都不會執行了
